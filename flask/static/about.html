How did we do it?


Convolutional Neural Networks is the hot new thing for machine learning. With the mass reduction in error rates for image processing due to CNN’s, CNN’s have become a powerful tool in today’s world. Companies all around the world use CNN’s including Google and even Instagram! But what are they?


Here we will briefly explain what they are using it’s most known and popular use case, image processing. Specifically how we used image processing to solve our digit recognition problem. This area will be for going into the more specifics of everything while the white font will be a broad overview.


Image to Machine


Convolutional Neural Nets: The first step was to set up the images so the NN could be able to read them. This is done by taking the Image and representing it as a matrix of values. In our case we took the image and created a matrix of it's pixel values.


In Depth: Since the computer cannot read in an image like we do images need to represented with pixel values. This is done by data preprocessing on the images to make sure the CNN is able to read the picture.

First Layer


The first layer of the CNN is always aConvolutional Layer. This layer acts like a flashlight that is shining over a region in the image. Then imagine the flahslight sliding across all the areas of the input image. In each are the flashlight shines over it takes a look to see if specific low level features are present in the area. In the end you end up with a matrix that states whether certain regions have a specific feature that a number is looking for.


In depth:The flashlight is called a filter and the part it is shining over is called the receptive field. The filters are called feature identifiers. Everytime the flashlight shines over a region it is multiplying the values in the filter by the original pixel values of the image. The weight of the filter is determined by what it's trying to look for, it is acting like a feature identifier. The multiplications are all summed up so at the end you have a unique number for the filter. This is then put into something called an activation map.


The In Between


Looking back at the Convolutional Layer we know that it is used for detecting low level features. But we need something that can detect high level features! So if we create more and more Convolutional Layers we can get higher and higher features.


In Depth:This is a tricky concept to grasp. When we go through another conv(Convolutional) layer the input is the activation map from the previous layer and so the flashlight is shining over a set of lower features trying to find a a feature from all the lower level features.

Fully Connected Layer


Now that we detected high level features we need something to make sense of them. This is where the Fully Connected Layer comes in. The fully connected layer is in charge of "choosing the right answer". It does this by giving a probability to each different possible outcome. For our project it was the digits 0 thru 9.


In depth:t gets all the information from all the previous layer and outputs a N dimensional vector with N being how many classes it can choose from, in our case it would be the numbers 0 thru 9. With each spot in the vector being the probability it is that class.
